# Output Templates

Use these templates for the two user-facing files. They are designed to be read quickly by the user: concrete state first, analysis second, audit details last.

## Review File

Filename pattern:

```text
codex_task_review_YYYY-MM-DD.md
```

Template:

```markdown
# Codex 未标记任务复盘（YYYY-MM-DD）

## 这次看到了什么

本次检查了 N 个未标记会话，排除当前控制线程后，实际需要处理 N 个。

一句话结论：
这里写最重要的判断，不超过 2 句话。

本轮会话主要分成几类：
- 类型 A：数量，代表会话，说明它为什么重要。
- 类型 B：数量，代表会话，说明它为什么重要。
- 类型 C：数量，代表会话，说明它为什么重要。

## 已经完成的事

| 任务 | 结果 | 是否还要跟进 |
|---|---|---|
| 任务名 | 已完成什么 | 否 / 是，原因 |
| 任务名 | 已完成什么 | 否 / 是，原因 |

补充说明：
这里只写那些表格放不下、但会影响判断的内容。

## 还没收口的事

| 任务 | 卡在哪里 | 下一步 |
|---|---|---|
| 任务名 | 卡点 | 明确动作 |
| 任务名 | 卡点 | 明确动作 |

如果没有未完成任务，写：
本轮没有发现需要继续推进的未完成任务。

## 值得保留的资产

这些会话不只是聊天记录，后续可能还会用到：

| 资产 | 为什么保留 | 位置 / 线索 |
|---|---|---|
| Skill / 自动化 / PRD / 代码改动 | 保留原因 | 会话名或文件路径 |

## 可以清理的会话

| 会话 | 建议 | 原因 |
|---|---|---|
| 会话名 | 删除 / 归档保留 | 原因 |

清理判断：
- 已总结、无后续价值：建议删除。
- 包含 Skill、自动化、PRD、业务决策、未完成工作：建议保留。
- 读不到或索引残留：记录原因后建议清理。

## 这次暴露的问题

只写 1-3 个真正影响效率的问题。

1. 问题：
   影响：
   下次怎么避免：

   影响：

   影响：
   下次怎么避免：

## 下次直接照做

```
